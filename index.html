Mysql查询索引失效的15种情况 select * 由于是全表扫描，不会走索引，建议写清楚那些字段，可能走覆盖索引（同联合索引） 联合索引最左匹配，联合索引的顺序和where中条件的顺序满足最左匹配，才会走索引 索引列参与了计算，索引会失效 索引列进行了函数操作 or会导致索引失效 in，not in等如果不是主键，都会失效 like，前导符是%会失效 is not null 索引失效，is null会正常走索引 索引列类型的隐式转换会失效 两列作比较会失效 不等于会失效 普通索引会查询结果集占比较大时索引会失效 orderBy会导致索引失效，有的版本不失效 参数不同导致索引失效 上述索引失效是因为DBMS发现全表扫描比走索引效率更高，因此就放弃了走索引。 也就是说，当Mysql发现通过索引扫描的行记录数超过全表的10%-30%时，优化器可能会放弃走索引，自动变成全表扫描。某些场景下即便强制SQL语句走索引，也同样会失效。
